# Changes 2026-05-12

## Scope
Update retrieval and assembly to align with the wrapper-first plan, remove BM25,
add planner-first CLI flow, and improve summary strategy with docstring seeding
and incremental updates.

## Changes
- Removed BM25 usage from build, storage, resolver, and confidence scoring.
- Added optional keyword fallback flag (default off) for entity misses.
- Refactored prompt assembly to use ModeSpec flags for layers and tests.
- Added docstring fallback for summaries in context assembly.
- Added docstring-seeded summaries at build time.
- Added incremental summarization for changed files on update.
- Added planner-first CLI option to call an SLM tool planner and expand context.
- Added error-only follow-up packets to avoid resending full context after failures.
- Added optional CLI flag to request docstring/comment updates.
- Wired `sg init` and `sg prepare` into the CLI entry point.
- Made docstrings the primary summary source with summary fallback.
- Added auto rebuild on completion for IDE and CLI when enabled.

## Reasons
- BM25 introduced noisy retrieval paths and conflicted with the intended
  model-driven target selection.
- ModeSpec alignment makes packet shape match the blueprint taxonomy.
- Docstring-first summaries reduce cost and improve early usability.
- Incremental summaries prevent large upfront costs on big repos.
- Planner-first CLI enables a two-step flow: plan tool calls, then execute.
- Error-only follow-ups reduce repeated context after failures.

## Notes
- Planner expansions are appended to the context packet for CLI runs.
- Keyword fallback stays disabled by default and can be enabled via config.
